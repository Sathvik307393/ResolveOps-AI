## ✨ What is Nexus AI?

Nexus AI is a multi-tenant **AI-powered Site Reliability Engineering (SRE) platform** deployed as a production-grade web application. It acts as your autonomous on-call engineer — continuously ingesting telemetry from your services, detecting anomalies, generating Root Cause Analysis (RCA) reports via a RAG-powered LLM engine, and dispatching targeted notifications — all from a stunning, glassmorphic Next.js dashboard.

---

## 🚀 Key Features

| Feature | Description |
|---|---|
| **🧠 AI Copilot** | LLM-powered chatbot that answers infrastructure questions, generates Kubernetes YAML, and analyzes your live logs using Retrieval-Augmented Generation (RAG). |
| **📊 Live Dashboard** | Real-time System Reliability Score computed from actual ingested log ratios. Dynamic sparkline charts update as new telemetry arrives. |
| **🚨 Incident Command Center** | Automatic incident creation when `ERROR` / `CRITICAL` logs are ingested. Displays RCA reports generated by the AI engine. |
| **🔍 Service Log Explorer** | Aggregates all services sending telemetry into a unified log terminal view, grouped by service name and severity. |
| **💡 DevOps Suggestion Hub** | Pre-mapped resolutions for common errors across the Development, Deployment, and Runtime stages of the software lifecycle. |
| **🔐 Multi-Tenant Auth** | JWT-secured user registration and login backed by AWS DynamoDB. Each user's data is fully isolated by tenant ID. |
| **📧 Email Notifications** | Automatic SES/SMTP email alerts dispatched the moment a new incident is created or an RCA is completed. |

```
                    └──────────────┬──────────────┬─────────────┘
                                   │              │
                       /api/*      │              │   /*
                                   ▼              ▼
                    ┌──────────────────┐  ┌──────────────────────┐
                    │  FastAPI Backend  │  │  Next.js 15 Frontend │
                    │   (Python 3.11)   │  │   (TypeScript + TW)  │
                    └────────┬─────────┘  └──────────────────────┘
                             │
              ┌──────────────┼──────────────────┐
              ▼              ▼                   ▼
     ┌──────────────┐  ┌──────────┐     ┌──────────────────┐
     │  AWS DynamoDB │  │  AWS SES │     │  Amazon Bedrock  │
     │  (3 tables)   │  │  Email   │     │  Claude 3 / RAG  │
     └──────────────┘  └──────────┘     └──────────────────┘
```

**Containers (Docker Compose):**
- `nexus-api` — FastAPI backend serving all `/api/*` routes
- `nexus-ui` — Next.js 15 production build serving the dashboard
- `nexus-nginx` — NGINX reverse proxy with SSL termination via Let's Encrypt

## 🛠️ Self-Hosted Deployment (AWS EC2)

### 1. Prerequisites

Create a `.env` file inside `nexus-aiops-bot/`:

```env
AWS_ACCESS_KEY_ID=your_aws_key
AWS_SECRET_ACCESS_KEY=your_aws_secret
AWS_REGION=us-east-1
JWT_SECRET=your_secure_random_string
```

> **Note:** Your AWS IAM user must have DynamoDB, SES, and Bedrock permissions.

### 2. Provision an EC2 Instance

Launch an Ubuntu 22.04 EC2 instance (t2.micro or larger). Open ports `80` and `443` in the Security Group.

### 3. Deploy with Docker Compose

SSH into your EC2 instance and run:

```bash
# Install Docker & Compose
sudo apt update && sudo apt install docker.io docker-compose -y
sudo systemctl start docker

# Clone the repository
git clone https://github.com/Sath2003/NexusAI.git
cd NexusAI/nexus-aiops-bot

# Add your environment variables
nano .env

# Build and launch all containers
sudo docker-compose up -d --build
```

### 4. SSL with Let's Encrypt (Optional but Recommended)

```bash
sudo apt install certbot -y
sudo certbot certonly --standalone -d your-domain.com
```

Then update `nginx/nginx.conf` with your domain name and restart the `nexus-nginx` container.

## 📡 Sending Telemetry to Nexus AI

Once your account is created, use your API key to send logs from any service:

```bash
curl -X POST https://nexusai.sathvikdevops.online/api/v1/ingest \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <your-jwt-token>" \
  -d '{
    "service": "payment-api",
    "level": "ERROR",
    "message": "Connection pool exhausted after 30s timeout",
    "latency_ms": 30045,
    "status_code": 503,
    "request_id": "req_abc123xyz"
  }'
```

When an `ERROR` or `CRITICAL` log is ingested, Nexus AI will:
1. Automatically create an incident in your dashboard
2. Trigger the AI RCA engine to analyze recent logs
3. Send you an email notification with the full incident report
